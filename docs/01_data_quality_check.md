# Проверка качества данных

## Цель проверки

Перед расчётом продуктовых метрик необходимо убедиться, что таблица пользователей содержит корректные данные и может использоваться как основной источник информации о зарегистрированных игроках.

В рамках проверки была выполнена оценка качества идентификаторов пользователей в таблице `players_profile`.

## Используемый запрос

Проверка выполнена запросом:

```text
sql/01_data_quality.sql
```

## Логика проверки

Для оценки качества данных были рассчитаны три показателя:

| Метрика                   | Для чего рассчитывается                                                          |
| ------------------------- | -------------------------------------------------------------------------------- |
| COUNT(*)                  | Определяет общее количество регистраций                                          |
| COUNT(player_id)          | Проверяет наличие записей без идентификатора игрока                              |
| COUNT(DISTINCT player_id) | Позволяет определить, существуют ли игроки, зарегистрированные более одного раза |

Если значения всех трёх показателей совпадают, это означает, что:

* все записи содержат идентификатор игрока;
* отсутствуют NULL-значения;
* отсутствуют дубликаты.

## Результаты проверки

| Метрика                          | Значение |
| -------------------------------- | -------: |
| Общее количество записей         |     3078 |
| Количество заполненных player_id |     3078 |
| Количество уникальных player_id  |     3078 |

## Вывод

Таблица `players_profile` содержит 3078 записей.

Проверка показала:

* все регистрации содержат идентификатор игрока;
* игроков без player_id не обнаружено;
* игроков, зарегистрированных более одного раза, не обнаружено.

Каждая строка таблицы соответствует одному уникальному игроку.

Данные могут быть использованы для дальнейшего расчёта продуктовых, маркетинговых и финансовых метрик без дополнительной очистки.
